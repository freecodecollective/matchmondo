# World Cup 2026 Fan Schedule

A single-page, no-build static website with the full FIFA World Cup 2026 schedule (June 11 – July 19, 2026 · 104 matches).

## Features

- **Time zone picker** — every kickoff is stored in UTC and rendered in whatever zone you choose (defaults to your device's zone). Choice is remembered.
- **Full schedule** — grouped by day, with stage/group, match number, opponents, venue, and city. Filter by stage or by team.
- **Show / hide scores** — toggle scores on or off (e.g., if you're avoiding spoilers while catching up on a recorded match).
- **Show FIFA ranking** — optional toggle that adds each team's current FIFA world-ranking position (`#N`) next to its name on every match and in the player guide. Data in `data/rankings.js`.
- **Show / hide US TV channel** — optional 📺 line per match for U.S. viewers.
- **Print / Save as PDF** — a clean print stylesheet strips the controls and colors. Optional **blank score boxes** mode prints empty boxes so you can fill in scores by hand.
- **Player guide** (optional, via the *Include player guide* checkbox) — top players for every team (5 each for the tournament favorites, 3 for the rest) with position, current club, hometown, and a short write-up. When included it renders below the schedule on screen and prints as its own section starting on a fresh page. Data lives in `data/players.js`, compiled from web research.
- **Google Calendar subscription** — **Subscribe in Google Calendar** points Google at the hosted `world-cup-2026.ics`, so it shows up as its own calendar (“World Cup 2026”) under *Other calendars*. You **show/hide it with a checkbox** (select/unselect) and Google re-fetches it periodically (`REFRESH-INTERVAL: 6h`) so scores stay current — no re-importing. Events are in UTC, so each subscriber sees kickoffs in their own time zone. A **Download .ics** button is also provided for one-time import (Apple Calendar, Outlook, offline).

  > **Subscription requires the site to be hosted at a public URL** — Google fetches the `.ics` from *its* servers, so it can't reach `localhost`. When served publicly the Subscribe button just works; locally, use Download → Import. The dialog detects localhost and says so.

## Running it

No build step. Open `index.html` directly, or serve the folder:

```sh
python3 -m http.server 8000
# then visit http://localhost:8000
```

## Updating scores

Edit `data/matches.js` — each match has `scoreH` / `scoreA` fields (set to `null` until played). The schedule, score toggle, and printouts all update automatically.

## Files

| File | Purpose |
|---|---|
| `index.html` | Page structure and controls |
| `styles.css` | Screen + print styles |
| `app.js` | Rendering, time zones, filters, printing, .ics generation |
| `data/matches.js` | All 104 matches (UTC kickoffs, venues, teams, scores) |
| `data/players.js` | Top players per team (name, position, club, hometown, blurb) |
| `data/rankings.js` | FIFA world-ranking position for each team (June 2026) |
| `world-cup-2026.ics` | Static subscribable calendar (regenerated by the updater) |
