How to review questions

  1. Click any question card to expand it and check the answer, options, and explanation.
  2. Edit any field directly — changes auto-save for everyone.
  3. Questions have three states: NOT READY (default), REVIEW (Claude made changes, needs human check), and READY (approved for users).
  4. Only questions marked READY will appear in the app.

Questions vs. Published

  • The Questions tab is your review queue — it shows only unpublished questions that need attention.
  • When you mark a question READY, it moves to the Published tab and goes live in the app.
  • To edit a published question, find it in the Published tab and click Edit — changes go live immediately.
  • To unpublish a question, edit it and change its status back to NOT READY — it will return to the Questions queue.

Controls

  • Drag handle (☰) — drag and drop to reorder questions
  • Arrow buttons — move a question up or down one spot
  • Pencil — open full edit modal
  • X — delete a question
  • + Add Question — create a new question from scratch
  • Save JSON — download a local copy of all questions

Task for Claude

Use the purple Task for Claude section to request changes (e.g. "Add 10 questions about penalty shootouts"). Claude will pick up the task, make changes, and mark affected questions as REVIEW so you can check them.

How saving works

Every change auto-saves to GitHub within 2 seconds. The green Synced badge in the header confirms your changes are live. If you see Sync failed, try refreshing the page.
