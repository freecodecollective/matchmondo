🏆 Pools — friend groups for Predictions & Trivia

Pools let anyone start a private competition: a name, a 6-char code, the games you want to play (Predictions / Trivia / both), and your own point values. Pre-kickoff, picks are hidden from everyone else in the pool — they unlock the moment the game starts. Forget to pick? You're auto-picked for the higher-FIFA-ranked team to win 1–0; if it hits, you only get the consolation 1 pt (not the full 3).

  • Anyone creates a pool. They share the code. People join with the code.
  • Pool creator sets name, type (Predictions / Trivia / Both), point values (defaults match how the app already scores), and a start date — invite people days early.
  • In Fun → Friends, both Leaderboard and Pools are zippable cards. If you're in multiple pools, each is its own row inside the Pools card.

Open questions for you

1. Pool size / cap? No cap implied — 100-person pools should "just work." Likely fine since picks and totals are cheap to compute. Confirm.

2. Pool start date for a pool that's already in flight (e.g. someone creates one on Jun 30 mid-group-stage) — current mock says points count from the chosen start date. If you want pools to be able to "include the whole tournament so far," that's a different toggle. Let me know.

3. Picks are global, not per-pool. Simpler implementation, less work for users (no picking the same match 3×) — but it means you can't strategically pick "BRA 5–0" in your office pool and "BRA 1–0" in your sibling pool. I think that's fine; flagging it because it's a load-bearing call.

4. "Both" pools split the leaderboard total or just sum predictions+trivia? Right now I'm summing (54p + 32t = 86). Could also tab them.

5. Auto-pick fallback uses FIFA ranking — what about knockouts where one team has already advanced from a less obvious path? FIFA rank still works but might pick the "wrong" favorite occasionally. Acceptable, I think.
